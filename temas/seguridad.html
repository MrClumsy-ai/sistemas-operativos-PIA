Redes y seguridad: sistemas distribuidos

Hazme click!

La seguridad en dispositivos móviles es un aspecto crítico en la era digital actual, donde la privacidad y la protección de datos son prioridades tanto para los usuarios como para los desarrolladores. Android, como una de las plataformas móviles más utilizadas en el mundo, ha implementado una serie de mecanismos y tecnologías avanzadas para garantizar la integridad, confidencialidad y autenticidad de los datos y aplicaciones.

Zona de pruebas para aplicaciones

Uno de los principios fundamentales de la seguridad en Android es el aislamiento de las aplicaciones. La plataforma aprovecha la protección basada en el usuario de Linux para asignar un ID de usuario único (UID) a cada aplicación. Este UID permite que cada app se ejecute en su propio proceso, creando una “zona de pruebas” (sandbox) a nivel de kernel. Este enfoque garantiza que los recursos de una aplicación estén aislados de los de otras aplicaciones, lo que reduce el riesgo de que una app maliciosa acceda o manipule datos de otras aplicaciones. Además, este mecanismo limita el impacto de posibles vulnerabilidades, ya que una brecha en una aplicación no compromete necesariamente al sistema en su totalidad.

Firma de Apps

La firma de aplicaciones es otro componente esencial de la seguridad en Android. Cada aplicación debe ser firmada digitalmente por su desarrollador antes de ser distribuida. Esta firma no solo identifica al autor de la aplicación, sino que también permite actualizaciones sin la necesidad de crear interfaces o permisos complicados. La firma digital asegura que la aplicación no ha sido alterada desde su publicación y que proviene de una fuente confiable. Este proceso es crucial para mantener la integridad del ecosistema de aplicaciones y prevenir la distribución de software malicioso.

Autenticación y Datos biométricos

La autenticación es un aspecto clave para proteger el acceso a los dispositivos y las aplicaciones. Android utiliza claves criptográficas controladas por la autenticación del usuario, que requieren un almacenamiento seguro y la participación de proveedores de servicios y autenticadores. En dispositivos con sensores de huellas dactilares, los usuarios pueden registrar una o más huellas para desbloquear el dispositivo o realizar transacciones seguras. El subsistema de Gatekeeper, que opera en un entorno de ejecución confiable (TEE), maneja la autenticación mediante contraseñas o patrones.

A partir de Android 9, se introdujo la API de Biometric Prompt, que permite a los desarrolladores integrar la autenticación biométrica de manera agnóstica en cuanto al dispositivo y la modalidad. Esto significa que las aplicaciones pueden utilizar huellas dactilares, reconocimiento facial o escaneo de iris sin necesidad de adaptarse a hardware específico. Además, sólo los datos biométricos seguros pueden integrarse en Biometric Prompt, lo que garantiza que la información sensible del usuario esté protegida.

Encriptación y Almacén de Claves

La encriptación es una herramienta fundamental para proteger los datos del usuario. En Android, una vez que se encripta un dispositivo, todos los datos creados por el usuario se encriptan automáticamente antes de almacenarse en el disco. De manera similar, las operaciones de lectura desencripta los datos antes de devolverlos al proceso de llamada. Este enfoque garantiza que, incluso si un atacante obtiene acceso no autorizado al dispositivo, no pueda leer la información almacenada.

Además, Android ofrece un almacén de claves respaldado por hardware, que proporciona funcionalidades avanzadas como la generación, importación y exportación de claves asimétricas, la encriptación y desencriptación con modos de padding adecuados, y más. Este almacén de claves es esencial para proteger las credenciales y los datos sensibles de las aplicaciones.

Security-Enhanced Linux (SELinux)

Android utiliza Security-Enhanced Linux (SELinux) para aplicar el control de acceso obligatorio (MAC) en todos los procesos, incluyendo aquellos que se ejecutan con privilegios de superusuario (sudo). SELinux es una extensión de seguridad de kernel de Linux que define políticas de acceso estrictas para prevenir que aplicaciones o procesos maliciosos comprometan el sistema, incluso si obtienen privilegios elevados.

Entorno de Ejecución Confiable (TEE)

El entorno de ejecución confiable (TEE) es otro pilar de la seguridad en Android. Trusty, un sistema operativo seguro, proporciona un TÉ que se ejecuta en el mismo procesador que Android, pero está aislado del resto del sistema mediante hardware y software. Este entorno se utiliza para realizar operaciones sensibles, como la autenticación biométrica y el manejo de claves criptográficas, en un espacio protegido de posibles ataques.

Inicio Verificado

El inicio verificado (Verified Boot) es una tecnología que garantiza que todo el código ejecutado en el dispositivo provenga de una fuente confiable, como el fabricante del dispositivo (OEM). Este proceso establece una cadena de confianza que comienza con una raíz de confianza protegida por hardware y se extiende hasta el bootloader, la partición de inicio y otras particiones verificadas. Si se detecta algún código no autorizado o corrupto, el dispositivo puede bloquearse para prevenir posibles ataques
